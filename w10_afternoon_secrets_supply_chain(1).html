Secrets Rotation

Secret plain text trong git?

AWS Secrets Manager + External Secrets Operator.
Rotate < 60s, không restart pod.

Problem

Secret hard-code trong manifest

DB password commit thẳng vào git. Ai clone repo = ai có password.

Password plain text trong git

DB_PASSWORD: "prod@123" commit vào repo. Git history giữ mãi. Leak 1 lần = toàn bộ lịch sử lộ.

Rotate cần restart pod

Đổi password trong Secret → pod phải restart mới thấy. Downtime 30s mỗi lần rotate. SLO vi phạm.

Không biết khi nào rotate

Password dùng 2 năm chưa đổi. Không có policy tự động rotate. Compliance audit fail.

ConfigMap dùng làm Secret

Dev dùng ConfigMap chứa password vì "tiện hơn Secret". Base64 không phải encryption.

Vấn đề gốc:
• K8s Secret chỉ là base64, không encrypt
• Rotate = restart pod

Lợi ích

ESO — Rotate < 60s, không restart pod

2 điểm mạnh: tự động reload + configurable refresh.

No restart needed

Pod mount Secret qua volume. ESO update Secret → volume tự reload. App đọc file mới.

Kết quả: Không cần restart pod. Zero downtime khi rotate.
refreshInterval

ESO poll AWS Secrets Manager mỗi refreshInterval (vd 60s). Thấy thay đổi → sync ngay.

W10 target: < 60s từ AWS rotate đến pod thấy mật khẩu mới.

Supply Chain Security

Image không rõ nguồn gốc?

Trivy scan trong CI + Cosign sign image + Admission verify signature.

Problem

Deploy image không biết ai build

Image build xong nhưng không ai kiểm: không scan CVE, không biết ai build, cụm cứ thế pull về chạy.

CI không scan CVE

Build xong push lên chạy luôn, không ai scan xem image có CVE (lỗ hổng bảo mật) nặng (HIGH/CRITICAL) không → production chạy lỗ hổng cả nửa năm mà không biết.

Image không được sign

Ai cũng push image lên registry. Cụm pull về mà không biết ai build, build lúc nào, có bị tamper (sửa lén) hay không.

Cluster không verify

Admission không check signature (chữ ký số của image) → dev push image từ máy cá nhân cũng chạy được, kẻ xấu chèn image độc rất dễ.

CVE chưa có patch

Có CVE nặng nhưng vendor chưa ra patch (bản vá) → CI fail chặn deploy luôn, mà không có quy trình xin exception (ngoại lệ) tạm thời.

Giải pháp: Trivy scan CVE trong CI (fail nếu HIGH/CRITICAL) + Cosign sign image + Admission chỉ cho chạy image đã sign.

Giải pháp

Supply Chain Flow

Trivy
Scan CVE trong CI, có CVE HIGH/CRITICAL thì fail. CVE chưa có patch thì xin exception (ghi ADR).
Cosign
Sign image bằng private key — signature lưu ngay trong registry, cạnh image. (2 kiểu: keyless OIDC hoặc key-based.)
Admission
Trước khi cho chạy, cụm verify signature. Image chưa sign hoặc signature không khớp → reject.

Trivy scan trong CI

GitHub Actions workflow với Trivy scan CVE.

1
Trivy Scan trong GitHub Actions
# .github/workflows/build.yml name: Build and Scan on: { push: { branches: [main] } } jobs: build: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - run: docker build -t myapp:${{ github.sha }} . # Trivy scan CVE - uses: aquasecurity/trivy-action@master with: image-ref: myapp:${{ github.sha }} severity: HIGH,CRITICAL exit-code: 1 # fail CI nếu có CVE HIGH+
02 — Supply Chain Security
Kết quả

Pass scan vs Fail scan

2 kịch bản: image clean hoặc có CVE HIGH/CRITICAL.

Pass scan
Image không có CVE HIGH/CRITICAL
Flow:
1. Trivy scan pass ✓
2. Tiếp tục bước 2 (Cosign sign)
3. Push image lên registry
4. Deploy được vào cluster
Fail scan
Image có CVE HIGH hoặc CRITICAL
Flow:
1. Trivy scan fail ✗
2. CI pipeline stop ngay
3. Không push image
4. Cần fix CVE hoặc viết exception ADR

Lab 2.1 · ESO

Đề bài: rotate secret không restart pod

App đang đọc DB password từ Secret plaintext. Chuyển sang AWS Secrets Manager + ESO tự sync → đổi giá trị trên AWS, K8s Secret cập nhật < 60s, pod không restart.

AWS credentials tạo bằng kubectl create secret — KHÔNG commit vào git.

Gợi ý — tạo file gì

eso/ ├── secret-store.yaml # SecretStore → AWS region + auth └── external-secret.yaml # map key AWS→K8s + refreshInterval argocd/apps/ ├── eso.yaml # App cài ESO operator (wave sớm) └── eso-config.yaml # App sync eso/ (wave SAU operator)

Gợi ý tư duy

  • refreshInterval: ngắn → spam AWS; dài → rotate chậm. Đặt sao để < 60s?
  • Vì sao đổi secret không cần restart pod? (mount volume vs env)
  • Thứ tự: ESO operator (CRD) phải có TRƯỚC khi apply SecretStore/ExternalSecret → tách 2 App + dùng sync-wave, đừng sync 1 lượt (lỗi no matches for kind SecretStore).
  • Không có AWS? fallback provider fake của ESO hoặc Sealed Secrets — vẫn chứng minh được pattern
